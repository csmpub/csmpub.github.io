동적 문서 생성을 위한 R 패키지 Knitr

코드 정크 옵션(Cell Options)

실행할 수 있는 프로그램 코드의 실행과 출력 관련한 설정을 제어하는 옵션으로 라인 맨 앞에 #|를 사용하여 지정한다.

속성(Attriutes)
label

코드쎌의 유일한 이름. 코드쎌을 fig-samples, tbl-summary 등에서 상호참조할 때 사용한다.

classes

코드쎌에 적용할 클라스.

renderings

렌더링 이름들의 배열(예시, [light, dark])

코드 출력(Code Output)
eval

코드쎌 실행 여부.

  • true (기본값): 실행
  • false: 실행하지 않음(코드 자체는 출력에 표시)
echo

소스코드를 렌더링 결과에 포함시킬지 여부.

  • true (대부분 포멧의 기본값): 포함
  • false (beamer, revealjs, pptx 포멧 등의 기본값): 제외
  • fenced: echo 옵션에 추가하여, 코드쎌 구분자(delimiter)를 출력에 포함.
code-fold
사용자가 필요할 때 펼처 볼 수 있도록 코드를 HTML

태그 안으로 펼쳐 넣을지 여부.

  • true: 펼쳐 넣음
  • false (기본값): 펼처 넣지 않음
  • show: 처음에 코드를 펼친 상태로 HTML
    태그 사용.
code-summary

code-fold를 사용한 경우 표시할 요약 문자열

code-overflow

코드가 길어서 코드쎌 폭을 넘어갈 때 처리 방법.

  • scroll: 좌/우 스크롤
  • wrap: 줄바꿈
code-line-numbers

코드블록 출력에 라인 번호 표시 여부 (true 또는 false).

힌트

revealjs 출력에서만 특정 라인을 강조(에니메이션 포함)할 수 있음.

여러 줄은 콤마(commas)로 표시:
  • 3,4,5
  • 1,10,12
범위는 대시(dashes)로 표시하고 콤마(commas)로 결함:
  • 1-3,5
  • 5-10,12,14
애니메이션 단계는 |로 분리:
  • 1-3|1-3,5 처음에 1-3, 그 다음에 1-3,5 라인
  • |5|5-10,12 처음에 번호 없음, 다음에 5, 그리고 나서 5-10 and 12 라인
lst-label

상호참조에 쓰이는 코드리스팅의 유일한 이름.

lst-cap

코드리스팅의 캡션

tidy

R 코드를 재정렬 할 지 여부.

tidy-opts

정렬 툴에 전달해 줄 옵션 리스트.

collapse

모든 소스와 출력 블록을 하나의 블록으로.

prompt

R 코드에 프롬프트를 포함할 지 여부.

class-source

소스코드 블록의 클라스 이름

attr-source

소스코드 블록의 속성

실행 결과 출력(Cell Output)
output

코드실행의 결과를 포함할 지 여부.

  • true: 포함
  • false: 제외
  • asis: 출력을 마크다운으로 간주(블록안에 별도로 취급하지 않음)
warning

실행 결과의 경고 포함 여부.

error

실행결과의 에러 포함 여부 (이 옵션 사용시 에러가 발생해도 코드 실행이 중지되지 않음).

include

모든 출력(코드 또는 실행결과) 포함 여부.

panel

출력 패널의 형식(tabset, input, sidebar, fill, center)

output-location

출력을 생성한 코드에 대한 출력 결과의 상대적인 위치. (revealjs에만 적용)

  • default: 코드 다음에 표시
  • fragment: In a fragment (not visible until you advance)
  • slide: 새로운 슬리이드
  • column: 인접한 칼럼
  • column-fragment: In an adjacent column (not visible until you advance)
message

메지지을 출력에 포함할 지 여부.

  • true: 포함.
  • false: 제외.
  • NA: 출력에 포함되지 않음(knitr 로그 콘솔에는 포함).
results

텍스트 (warnings, messages, errors 아닌 일반 텍스트) 결과 표시 지정.

  • markup: 출력 결과를 코드블록 안에 표시
[1] 1 2 3
  • asis: 출력 결과를 다른 마크업 없이 문서로 바로 출력
cat("I'm raw **Markdown** content.\n")

I’m raw Markdown content.

  • hold: 텍스트 출력 결과를 보류 했다가 끝으로 모아서 보여줌

  • hide (또는 false): 텍스트 출력 감춤

comment

텍스트 출력물 앞에 표시할 코멘트 문자. (기본값 ##, 없애려면 '' 지정)

class-output

텍스트/콘솔 출력의 클라스 이름

attr-output

텍스트/콘솔 출력의 속성

class-warning

경고 출력의 클라스 이름

attr-warning

경고 출력의 속성

class-message

메시지 출력의 클라스 이름

attr-message

메시지 출력의 속성

class-error

에러 출력의 클라스 이름

attr-error

에러 출력의 속성

그림(Figures)
fig-width

그림 폭 기본값

fig-height

그림 높이 기본값

fig-cap

그림 제목

fig-subcap

그림 부제목

fig-link

그림의 링크(Hyperlink target)

fig-align

그림 수평 정렬 (default, left, right, 또는 center)

fig-alt

HTML 이미지의 대체 속성으로 쓰일 대체 텍스트

fig-env

그림 출력을 위한 LaTeX 환경

fig-pos
  • \begin{figure}[]에 사용될 LaTeX 그림 위치 정렬
  • 코드로 생성된 그림은 fig-pos="H"로 지정
  • fig-pos가 false인 경우 그림 위치 지정자(figure position specifier)는 사용되지 않음(sidewaysfigure로 설정된 경우에 필요)
fig-scap

LaTeX 출력에만 사용될 짧은 제목(\caption[]에 삽입되어 일반적으로 PDF 문서의 그림목차 “List of Figures” 에 표시)

fig-format

그림 출력 기본값(retina, png, jpeg, svg, 또는 pdf)

fig-dpi

그림 DPI 기본값

fig-asp

출력(plot)의 가로/세로 비율(fig-asp이 지정되면 fig-height = fig-width * fig-asp로 계산됨)

out-width
  • 출력 문서에서 그림(plot)의 넓이(출력 문서에서 축척으로 반영)
  • 사용할 수 있는 단위(생략할 경우 pixels)는 px, cm, mm, in, inch, %
out-height
  • 출력 문서에서 그림(plot)의 높이(출력 문서에서 축척으로 반영)
  • 출력 형식에 따라 특별한 값을 가질 수 있음(LaTeX: 3in, or 8cm, HTML: 300px)
fig-keep

코드 청크에서 출력(plot)이 어떻게 유지될 지 결정

  • high: high-level plots 유지 (low-level plot 변경은 high-level plots에 병합)
  • none: 모든 plots 무시
  • all: 모든 plots 유지 (low-level plot 변경은 새로운 plots 생성).
  • first: 첫번째 plot만 유지
  • last: 마지막 출력만 유지
  • 수치 벡터(numeric vector): 수치 벡터의 값은 유지할 출력(plot)의 인덱스들
fig-show

플롯을 어떻게 보여주고 배치할 것인지 지정

  • asis: plots이 생성된 위치에 유지(코드가 R 터마널에서 실행될 경우 표시될 위치)
  • hold: 모든 출력(plots and output)을 보류 하였다가 코드 청크의 맨 끝에 표시
  • animate: 코드 청크의 출력이 다수일 경우 모든 출력을 모아서 애니메이션으로 표시
  • hide: 출력 파일(plot files)은 생성하지만 출력 문서에서는 감춤
out-extra

그림에 적용될 추가 raw LaTeX 또는 HTML 옵션

external

tikz graphics 외면화(externalize), pre-compile to PDF

sanitize

tikz graphics 분리(sanitize), escape special LaTeX characters

interval

에니메이션 프레임 간 시간 간격(초)

aniopts

에니메이션에 적용할 추가 옵션들(LaTeX animate package 문서 참조)

animation-hook

HTML 출력에 에니메이션을 만들 함수(hook function)

  • 기본함수 ffmpeg는 FFmpeg 이미지를 WebM 비디오로 바꿀 때 사용
  • gifski package의 함수 gifski는 GIF 에니메이션을 생성할 때 사용
표(Tables)
tbl-cap

표 제목

tbl-subcap

표 부제목

tbl-colwidths

표(markdown grid tables) 열의 넓이를 명시적으로 지정(HTML 형식의 경우 지정할 필요 없으나 LaTex 형식 등에서 지정 필요)

  • auto: 표 안에 하이퍼링크가 있는 경우를 제외하고 markdown table column 적용(HTML 출력 형식의 기본값)
  • true: markdown table widths 적용(HTML 출력 형식이 아닌 모든 경우 기본값)
  • false: markdown table widths 적용하지 않음
  • 수치 배열 (e.g. [40, 30, 30]): 폭의 퍼센트 배열
html-table-processing

none으로 지정하면 raw HTML table을 처리하지 않고 그냥 둠(as-is)

패널 배열(Panel Layout)
layout

  • 열, 행의 2차원 배열로 지정
    처음 두개의 출력 블록을 옆으로 배치하고, 세번쩨 블록을 아래쪽에 전체 넓이로 배치하는 경우: [[3,3], [1]]

  • 마진을 생성하려면 음수 사용
    위 예에서 위에 있는 두 블록 사이에 마진을 두려는 경우: [[3,-1, 3], [1]]

layout-ncol

출력 블록을 열로 배열

layout-nrow

출력 블록을 행으로 배열

layout-align

출력 내용 수평 정렬(default, left, right, or center)

layout-valign

출력 내용 수직 정렬(default, top, center, or bottom)

페이지 열(Page Columns)
column

출력이 배치될 Page column

fig-column

그림 출력이 배치될 Page column

tbl-column

표 출력이 배치될 Page column

cap-location

표 제목의 위치 (top, bottom, or margin)

fig-cap-location

그림 제목의 위치 (top, bottom, or margin)

tbl-cap-location

표 제목의 위치 (top, bottom, or margin)

캐시(Cache)
cache

Whether to cache a code chunk. When evaluating code chunks for the second time, the cached chunks are skipped (unless they have been modified), but the objects created in these chunks are loaded from previously saved databases (.rdb and .rdx files), and these files are saved when a chunk is evaluated for the first time, or when cached files are not found (e.g., you may have removed them by hand). Note that the filename consists of the chunk label with an MD5 digest of the R code and chunk options of the code chunk, which means any changes in the chunk will produce a different MD5 digest, and hence invalidate the cache.

cache-vars

Variable names to be saved in the cache database. By default, all variables created in the current chunks are identified and saved, but you may want to manually specify the variables to be saved, because the automatic detection of variables may not be robust, or you may want to save only a subset of variables.

cache-globals

Variables names that are not created from the current chunk.

This option is mainly for autodep: true to work more precisely—a chunk B depends on chunk A when any of B’s global variables are A’s local variables. In case the automatic detection of global variables in a chunk fails, you may manually specify the names of global variables via this option. In addition, cache-globals: false means detecting all variables in a code chunk, no matter if they are global or local variables.

cache-lazy

Whether to lazyLoad() or directly load() objects. For very large objects, lazyloading may not work, so cache-lazy: false may be desirable (see #572).

cache-rebuild

Force rebuild of cache for chunk

cache-comments

Prevent comment changes from invalidating the cache for a chunk

dependson

Explicitly specify cache dependencies for this chunk (one or more chunk labels)

autodep

Detect cache dependencies automatically via usage of global variables

code

String containing code to execute for this chunk
